# Plan: #44 — Semantic search layer (embeddings + RRF fusion)

**Branch:** `feat/44-semantic-search` · **Worktree:** `../wt-44` · **Base:** `feat/blog-search-rebuild` (post-#43 tip) · Wave 3 (depends on #43, merged)

## Goal

Add semantic search on top of #43's FTS so conceptual queries with no keyword overlap still find the right post (OpenAI `text-embedding-3-small`, 1536 dims, via Vercel AI SDK). Index-time: chunk + embed each post into `PostChunk`, guarded so unchanged posts make **zero** OpenAI calls on re-runs. Query-time: embed the query, rank posts by best-chunk cosine distance, and fuse with the FTS ranking via **Reciprocal Rank Fusion (JS-side, k=60)**. Must degrade gracefully to FTS-only (no key / short query / embed fails/times out) and never hard-fail. Build/deploy must succeed with **no OpenAI key and unreachable DB**.

## Critical guardrails

- **NEVER point `DATABASE_URL` at production.** Embedding is folded into `blog:index`, so running the indexer now hits OpenAI *and* writes to whatever `DATABASE_URL` targets. All DB verification uses a **throwaway `pgvector/pgvector:pg16` Docker container** (plain `postgres:16` lacks the extension). The prod `CREATE EXTENSION vector` + `migrate deploy` are deploy-time human steps.
- Real embedding calls cost ~pennies for the whole corpus — fine against the throwaway DB, never against prod.

## Key resolved decisions (orchestrator)

1. **Fusion: (A) JS-side RRF.** `searchPostIds` (FTS, #43) + `searchPostIdsSemantic` (new) each return full ranked lists; a pure `reciprocalRankFusion([fts, semantic], 60)` merges them; the fused id list feeds the **existing** Prisma `where` composition (filters stay in Prisma — no fragile raw-SQL `_PostToTag` join). Not the issue's "single $queryRaw two-CTE" phrasing — that's an implementation preference, and (A) satisfies every AC while keeping RRF unit-testable and degradation trivial.
2. **Re-embed guard: `PostChunk.contentHash`** (the parent `Post.content` sha256 snapshot at embed time, denormalized onto each chunk row). NOT a new `Post` column — keeps #44's bookkeeping in its own table, respects the `Post.contentHash` comment ("#44 reads"). Guard: `FORCE_REEMBED || no chunks for post || chunk.contentHash !== post.contentHash`. Crash-safe: chunks are written atomically in a transaction opened *only after* `embedMany` succeeds; any failure leaves old chunks + old hash → detected and retried next run.
3. **Embedding folded into `blog:index`** (per-post loop in `blog-indexer.ts`), not a separate `blog:embed` script — matches "index-time embeddings" and gives #45 a single command.
4. **API key passed as an argument** to embedding functions. App/query-time caller (`blog.ts`) passes `env.OPENAI_API_KEY`; indexer script passes `process.env.OPENAI_API_KEY` (mirrors the `DEFAULT_AUTHOR_ID`/`FORCE_REEMBED` process.env precedent). This decouples the indexer/CI from full t3-env validation — #45's Action then needs only `DATABASE_URL` + `OPENAI_API_KEY` (+ optional `DEFAULT_AUTHOR_ID`), NOT the Discord/Twitch/Auth secrets.
5. **Semantic query bounded** `LIMIT 50 ORDER BY distance ASC` so pagination `total` stays meaningful as the corpus grows (harmless at ~11 posts).

### 3. Pure helpers in `src/lib/` (DB-free, network-free, ~/env-free, unit-tested)
- **`chunk-text.ts`** — `chunkText(text, {maxChars=1500, overlap=200}): {index,content}[]`. Boundary-preference hierarchy paragraph(`\n{2,}`)→`\n`→sentence(`. ! ?`)→whitespace→hard cut. **Note:** `Post.content` is currently a flat single-spaced string (the #43 extractor collapses all whitespace, `blog-content-extractor.ts:153`), so "paragraph-aware" degrades to sentence/word/hard split today — DO NOT modify the extractor (out of scope; would rehash every post). Clamp `step = Math.max(1, maxChars - overlap)` (prevents infinite loop when overlap≥maxChars). Empty/whitespace → `[]`; shorter than maxChars → 1 chunk.
- **`reciprocal-rank-fusion.ts`** — `reciprocalRankFusion(lists: {id}[][], k=60): {id,score}[]`. `score = Σ 1/(k + rank)` (1-indexed rank per list); ids in only one list still score (this is what makes a zero-keyword-overlap conceptual query surface via the semantic list); sort by score desc (V8 stable sort → deterministic ties).
- **`lru-cache.ts`** — tiny `LruCache<K,V>(maxSize)` over a `Map` (insertion order = LRU; `get` bumps recency; evict first key over capacity). maxSize ~50.
- **`vector-literal.ts`** — `toVectorLiteral(v: number[]): string` → `"[" + v.map(fmt).join(",") + "]"`. **Guard against scientific notation** (pgvector text parser: avoid `1e-8`-style output; use a formatter that emits plain decimals, e.g. reject/`toFixed`-style, and unit-test tiny/negative values).

### 4. Embedding module `src/server/embeddings.ts` (imports the AI SDK; key passed in, NOT read from ~/env here)
- `getEmbeddingModel(apiKey: string | undefined)` → `null` if no key, else `createOpenAI({ apiKey }).embedding("text-embedding-3-small")`. **Use `createOpenAI` explicitly, never the ambient `openai` singleton** — so the client is never constructed when the key is absent (build-without-key + import-never-throws are then trivially true; construction happens lazily inside the function, never at module top level).
- `embedQuery(q, apiKey): Promise<number[] | null>` — check module-level LRU (key = `q.trim().toLowerCase()`); if no model → null; else `embed({ model, value: q, abortSignal: AbortSignal.timeout(2000) })`, cache + return `embedding`; **catch all → null** (timeout/network/rate-limit degrade, never throw).
- `embedChunks(values: string[], apiKey): Promise<number[][] | null>` — `embedMany({ model, values })`; catch → null. (`embedMany` has no partial-success mode — a failed batch throws; caller catches per-post.)

### 5. Schema — `prisma/schema.prisma`
```prisma
model Post {
  // ...unchanged #43 fields...
  chunks PostChunk[]
}

model PostChunk {
  id          String                      @id @default(cuid())
  postId      String
  post        Post                        @relation(fields: [postId], references: [id], onDelete: Cascade)
  chunkIndex  Int
  content     String                      @db.Text
  contentHash String                      // sha256(hex) of parent Post.content at embed time — #44 re-embed guard
  embedding   Unsupported("vector(1536)")
  createdAt   DateTime                    @default(now())

  @@unique([postId, chunkIndex])
  @@index([postId])
}
```
Keep `embedding` `Unsupported` **plain** (no default/expr in the type string — Prisma bug #24496, same rule as #43's `searchVector`). `Unsupported` fields are excluded from `PostCreateInput`/`PostUpdateInput`/default select, so existing `Post` queries are unaffected and `embedding` can only be written via raw SQL. `pnpm db:generate` must succeed offline (proven by #43's `searchVector`).

### 7. Indexer — `src/scripts/blog-indexer.ts` (fold embedding into the per-post loop, after the existing `post.upsert`)
Read `const forceReembed = !!process.env.FORCE_REEMBED;` and `const apiKey = process.env.OPENAI_API_KEY;` (both raw process.env, like `DEFAULT_AUTHOR_ID`). After upserting each post (which now has `id`, `content`, `contentHash` in scope), call an `embedPostChunks(prisma, post, apiKey, forceReembed)` helper (put it in the script or a small module the script imports — must NOT force ~/env on the script):
```
if (!content) {                       // content removed/empty → cleanup, no API call
  await prisma.postChunk.deleteMany({ where: { postId: post.id } });
  return;
}
const existing = await prisma.postChunk.findFirst({ where: { postId: post.id }, select: { contentHash: true } });
const needs = forceReembed || !existing || existing.contentHash !== contentHash;
if (!needs) return;                   // ZERO OpenAI calls on unchanged re-run
if (!apiKey) { console.warn(`skip embed ${slug}: no OPENAI_API_KEY`); return; }  // leave state stale → retried when key present
const chunks = chunkText(content);
const embeddings = await embedChunks(chunks.map(c => c.content), apiKey);
if (!embeddings) { console.warn(`skip embed ${slug}: embed failed`); return; }   // leave old chunks intact (better than deleting)
await prisma.$transaction(async (tx) => {
  await tx.postChunk.deleteMany({ where: { postId: post.id } });
  for (let i = 0; i < chunks.length; i++) {
    await tx.$executeRaw(Prisma.sql`INSERT INTO "PostChunk" ("id","postId","chunkIndex","content","contentHash","embedding")
      VALUES (${randomUUID()}, ${post.id}, ${chunks[i].index}, ${chunks[i].content}, ${contentHash}, ${toVectorLiteral(embeddings[i])}::vector)`);
  }
});
```
- Only the INSERT needs raw SQL (deleteMany/findFirst are plain Prisma — `contentHash`/`chunkIndex`/`content` are ordinary scalars). Use `randomUUID()` from `node:crypto` (raw INSERT bypasses `@default(cuid())`). Bind the vector literal as a normal `${}` param + `::vector` cast (injection-safe, same mechanism as #43's `${q}`).
- Wrap the per-post embed in try/catch so one bad post doesn't abort the run (`embedMany` is all-or-nothing per post).
- The transaction opens **after** `embedChunks` succeeds — never hold a DB txn across the network call. Consider `{ timeout }` on `$transaction` if chunk counts grow.

### 8. Query-time — `src/server/api/routers/blog.ts`
Add `searchPostIdsSemantic(db, queryEmbedding): Promise<{id,distance}[]>` next to `searchPostIds`:
```sql
SELECT "PostChunk"."postId" AS id, MIN("PostChunk"."embedding" <=> ${toVectorLiteral(emb)}::vector) AS distance
FROM "PostChunk" JOIN "Post" ON "Post"."id"="PostChunk"."postId"
WHERE "Post"."published" = true
GROUP BY "PostChunk"."postId" ORDER BY distance ASC LIMIT 50
```
(`<=>` = pgvector cosine distance, matches `vector_cosine_ops`.) In `getAll`, weave semantic in behind the existing composition, output shape UNCHANGED:
```
const ftsIds = q ? await searchPostIds(ctx.db, q) : null;
let rankedIds = ftsIds;
if (q && q.trim().length >= 3) {
  const emb = await embedQuery(q, env.OPENAI_API_KEY);        // never throws; null on no-key/short/timeout/error
  if (emb) {
    try {
      const sem = await searchPostIdsSemantic(ctx.db, emb);
      rankedIds = reciprocalRankFusion([ftsIds!, sem]).map(({id,score}) => ({ id, rank: score }));
    } catch { /* pgvector/table missing → keep FTS-only */ }
  }
}
// use `rankedIds` exactly where #43 used it (Prisma where id:{in}, count({where}), reorderByRank(...).slice)
```
`reorderByRank` only reads array order (rank value unused) → fused `{id, rank}` feeds in with no adapter. Degradation branch map (all → FTS-only, none can throw out of getAll): no `q`; `q<3`; no key; embed throws/times out; semantic query throws; empty semantic list (RRF degenerates to FTS order). `import { env } from "~/env"` at top of blog.ts (app-side, validated).
