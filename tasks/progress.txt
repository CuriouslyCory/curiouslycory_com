## Codebase Patterns
- Auth.js v5 uses `export const { auth, handlers, signIn, signOut } = NextAuth({ ... })` pattern
- Auth.js v5 session access: use `auth()` instead of `getServerSession(authOptions)`
- Zod v4 removed `z.preprocess()` — use `z.unknown().transform().pipe()` instead
- blog-indexer scripts use `tsx` to run TypeScript outside Next.js (with dotenv/config for env loading)
- SSL connection string handling: strip `sslmode=` params before passing to PrismaPg adapter
- Next.js 16 removed `next lint` command - use `eslint .` directly instead
- eslint-config-next 16 exports native flat config - no FlatCompat needed
- React compiler rules (eslint-plugin-react-compiler) ship with eslint-config-next 16 - avoid setState in useEffect, use derived state or useSyncExternalStore instead
- For mount detection, use `useSyncExternalStore` with getServerSnapshot returning false (see `src/hooks/use-mounted.ts`)
- Use `useState(() => ...)` initializer for random values that should be computed once (not useMemo with impure functions)
- Typecheck command is `npx tsc --noEmit` (no `pnpm typecheck` script defined)
- pnpm overrides for @types/react and @types/react-dom must be kept in sync with devDependencies versions
- Prisma v7 uses `provider = "prisma-client"` (not `prisma-client-js`) and requires explicit `output` in generator block
- Prisma v7 removes `url` from datasource block — connection URL goes in `prisma.config.ts` instead
- Import PrismaClient from generated path (`~/generated/prisma/client.js`), not from `@prisma/client`
- `@prisma/adapter-pg` is needed for Direct TCP connections in Prisma v7
